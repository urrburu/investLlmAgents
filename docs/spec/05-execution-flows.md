# 대표 실행 흐름

## Flow A · LLMwiki 갱신

1. **원문 입력** — 책, 매매일지, 리포트, 메모 업로드
2. **RAG 처리** — 파싱 → 청킹 → 임베딩 → vector 저장
3. **지식 추출** — 원칙, 실수 패턴, 종목 thesis 추출
4. **위키 초안** — 새 페이지 / 수정 / 병합 제안 생성
5. **검증/승인** — citation 검증 후 저장 또는 승인 대기

### Shadow paths

- 원문 파싱 실패: 문서 단위로 실패를 기록하고 다른 문서 처리를 계속한다.
- citation 부족: 위키 revision을 저장하지 않고 `needs_human_review`로 둔다.
- 기존 페이지와 충돌: 병합하지 않고 충돌 요약과 후보 revision을 생성한다.

## Flow B · 정기 포트폴리오 점검

1. **스케줄 실행** — 일간/주간 자동 트리거
2. **현재 상태 계산** — 비중, 손익, 변동, 집중도 계산
3. **위키 참조** — 책 원칙, 매매 실수, 기존 thesis 검색
4. **충돌 감지** — 원칙과 현재 포트폴리오의 불일치 탐지
5. **리포트 생성** — 추천 없이 현황과 확인 항목 정리

### Shadow paths

- 가격 데이터 누락: 해당 종목의 수익률 계산을 보류하고 누락 상태를 표시한다.
- 포트폴리오 입력 오류: 파일/필드 오류를 반환하고 추정 계산을 하지 않는다.
- 원칙 충돌이 약함: 확정 표현 대신 "확인 필요" 질문으로 낮춰 표시한다.

## Flow C · 시황분석 · Risk-on/Risk-off 판단

1. **시장 데이터 수집** — 지수, 금리, 환율, 원자재, 변동성, 섹터 흐름 수집
2. **매크로/수급 점검** — VIX, 미 국채금리, 달러, 신용스프레드, 섹터 로테이션 확인
3. **Risk-on/off 판단** — 위험자산 선호/회피 국면을 rule + LLM 해석으로 분류
4. **포트폴리오 영향 연결** — 현재 보유 섹터, 성장주/방어주, 현금 비중과 연결
5. **시황 브리프 생성** — 오늘의 시장 국면, 근거, 확인할 리스크를 요약

### Shadow paths

- 신호가 엇갈림: `risk_on`, `risk_off`로 단정하지 않고 `mixed`로 표시한다.
- 데이터가 오래됨: 국면 판단을 차단하고 최신 데이터 확인 요청을 반환한다.
- 포트폴리오 연결 실패: 시황 브리프만 생성하고 포트폴리오 영향은 누락 사유를 표시한다.

## Risk-on / Risk-off 판단 신호

### Risk-on 신호

- 주가지수 상승과 시장 breadth 개선
- VIX 하락 또는 안정
- 하이일드 / 성장주 / 경기민감 섹터 강세
- 신용스프레드 축소
- 달러 약세 또는 위험통화 강세

### Risk-off 신호

- 주가지수 하락과 방어 섹터 상대 강세
- VIX 급등 또는 변동성 확대
- 미 국채, 달러, 금 등 안전자산 선호
- 신용스프레드 확대
- 유동성 / 금리 / 지정학 리스크 부각

## Flow D · 종목 Snapshot

1. **요청 정규화** — ticker, 거래소, 회사명을 정리한다.
2. **데이터 수집** — 가격, 재무, 뉴스, 공시를 조회한다.
3. **위키 연결** — 기존 thesis, 리스크, 관련 투자 원칙을 검색한다.
4. **변화 요약** — thesis에 영향을 줄 수 있는 이벤트와 확인 질문을 정리한다.
5. **검증** — 숫자와 출처를 확인하고 금지 표현을 제거한다.

### Shadow paths

- ticker가 모호함: 후보 목록을 반환하고 분석을 보류한다.
- 뉴스만 있고 공시가 없음: 뉴스 기반 주장임을 명시하고 확정 표현을 제한한다.
- 기존 thesis가 없음: Snapshot은 만들되 새 thesis 초안 생성을 제안한다.

## Flow E · 검증 루프

1. **숫자 검증** — 계산 결과와 원천 데이터를 대조한다.
2. **출처 검증** — 모든 주요 주장에 source ref가 있는지 확인한다.
3. **표현 검증** — 매수/매도 지시, 목표가, 보장 표현을 제거한다.
4. **품질 점수** — 충분성, 일관성, 최신성을 점수화한다.
5. **승인/보류** — 통과하면 포맷팅으로 보내고, 실패하면 수정 요청을 반환한다.
