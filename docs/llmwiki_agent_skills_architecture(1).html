Personal Investment LLMwiki · Agent · Skills 정리

LLMwiki와 Agent Skills를 분리한
개인 투자 점검 시스템 구조

이 문서는 책의 투자 원칙, 매매일지, 현재 포트폴리오를 기반으로 한 LLMwiki와 정기 점검/종목 분석/포트폴리오 분석을 수행하는 LangGraph Agent, 그리고 각 Agent가 호출하는 Skills를 명확히 구분한다.

1. 한 문장 정의

LLMwiki는 “나의 투자 지식 저장소”이고, Agents는 “그 지식을 사용해 판단 흐름을 실행하는 작업자”이며, Skills는 “Agent가 호출하는 개별 기능”이다.

2. 세 계층의 책임 분리

LLMwiki · Knowledge Layer

무엇을 저장하나

책, 매매일지, 현재 포트폴리오, 과거 리포트, 종목별 메모를 구조화해서 저장한다.

  • 투자 원칙
  • 내 반복 실수
  • 종목별 thesis
  • 포트폴리오 현황
  • 출처와 근거
Agents · Workflow Layer

무엇을 실행하나

LangGraph로 정기 점검, 수동 종목 분석, 포트폴리오 분석, 위키 갱신 흐름을 실행한다.

  • 정기 현황 점검
  • 종목 분석 요청 처리
  • 포트폴리오 분석 요청 처리
  • 책/일지/포트폴리오 연결
  • 검증 후 리포트 생성
Skills · Tool Layer

무엇을 호출하나

Agent가 필요할 때 호출하는 작은 기능 단위다. 계산, 검색, 청킹, 임베딩, 검증, 포맷팅이 여기에 속한다.

  • chunking
  • embedding
  • price fetch
  • portfolio metrics
  • citation verification

5. LLMwiki의 정보 구조

위키 네임스페이스

/wiki/books /principles /risk-management /position-sizing /psychology /wiki/journal /winning-patterns /mistake-patterns /emotion-driven-trades /postmortems /wiki/portfolio /current /allocation /risk-exposure /change-history /wiki/assets /{ticker} /{ticker}/thesis /{ticker}/events /{ticker}/risks /wiki/rules /buy-checklist /sell-checklist /rebalance-rules /invalidation-rules

각 페이지가 담아야 하는 것

  • 정의: 이 개념/종목/원칙이 무엇인지
  • 출처: 책, 매매일지, 리포트, 뉴스, 공시
  • 내 투자와 연결: 내 매매 패턴 또는 현재 포트폴리오와의 관련성
  • 충돌/위험 신호: 원칙과 현재 상태가 어긋나는 지점
  • 확인할 항목: 다음 점검 때 다시 볼 질문
중요: 위키는 “추천을 저장하는 공간”이 아니라 “근거, 원칙, 관찰, 복기, 현재 상태를 연결하는 공간”이다.

6. Agent 종류와 역할

Agent Trigger 읽는 지식 호출하는 Skills 출력
LLMwiki 갱신 Agent 책/일지/문서 업로드 기존 위키, 새 원문 파싱, 청킹, 임베딩, entity 추출, citation 검증 위키 페이지 생성/수정 초안

검증 Agent 모든 리포트 생성 후 계산 결과, 원문 chunk, citation 숫자 검증, citation 검증, 금지 표현 검사 검증 통과/보류/수정 요청
Persona Formatter Agent 최종 응답 직전 검증된 분석 결과 스타일 변환, 길이 조정, 출력 포맷팅 존나조쿤/떠들이 등 스타일화된 답변

8. 대표 실행 흐름 A: LLMwiki 갱신

1
원문 입력 책, 매매일지, 리포트, 메모 업로드
2
RAG 처리 파싱 → 청킹 → 임베딩 → PostgreSQL/pgvector 저장
3
지식 추출 원칙, 실수 패턴, 종목 thesis 추출
4
위키 초안 새 페이지/수정/병합 제안 생성
5
검증/승인 citation 검증 후 저장 또는 승인 대기

9. 대표 실행 흐름 B: 정기 포트폴리오 점검

1
스케줄 실행 일간/주간 자동 트리거
2
현재 상태 계산 비중, 손익, 변동, 집중도 계산
3
위키 참조 책 원칙, 매매 실수, 기존 thesis 검색
4
충돌 감지 원칙과 현재 포트폴리오의 불일치 탐지
5
리포트 생성 추천 없이 현황과 확인 항목 정리

10. 대표 실행 흐름 C: 시황분석 · Risk-on/Risk-off 판단

1
시장 데이터 수집 지수, 금리, 환율, 원자재, 변동성, 섹터 흐름 수집
2
매크로/수급 점검 VIX, 미 국채금리, 달러, 신용스프레드, 섹터 로테이션 확인
3
Risk-on/off 판단 위험자산 선호/회피 국면을 rule + LLM 해석으로 분류
4
포트폴리오 영향 연결 현재 보유 섹터, 성장주/방어주, 현금 비중과 연결
5
시황 브리프 생성 오늘의 시장 국면, 근거, 확인할 리스크를 요약

Risk-on 판단에 쓰는 대표 신호

  • 주가지수 상승과 시장 breadth 개선
  • VIX 하락 또는 안정
  • 하이일드/성장주/경기민감 섹터 강세
  • 신용스프레드 축소
  • 달러 약세 또는 위험통화 강세

Risk-off 판단에 쓰는 대표 신호

  • 주가지수 하락과 방어 섹터 상대 강세
  • VIX 급등 또는 변동성 확대
  • 미 국채, 달러, 금 등 안전자산 선호
  • 신용스프레드 확대
  • 유동성/금리/지정학 리스크 부각

12. 추천/비추천이 아니라 현황 점검으로 제한

허용되는 출력

  • 현재 포트폴리오 구성과 변화
  • 종목별 최근 이벤트 요약
  • 책 원칙과 현재 행동의 충돌 가능성
  • 과거 매매 실수와 유사한 패턴
  • 확인해야 할 질문 목록

제한해야 하는 출력

  • 지금 매수/매도하라는 직접 지시
  • 목표가/수익률 보장
  • 자동매매 실행
  • 근거 없는 확정적 전망
  • 출처 없는 투자 판단

14. 구현 우선순위

1
데이터 모델 책, 매매일지, 포트폴리오, 위키 페이지 스키마
2
RAG 파이프라인 청킹, 임베딩, 검색, citation 연결
3
위키 갱신 Agent 원칙/패턴 추출과 페이지 revision 생성
4
포트폴리오 점검 Agent 계산 + 위키 참조 + 리포트 생성
5
검증/스타일 숫자/출처 검증 후 GPT 스타일 포맷팅
핵심 원칙: 숫자는 코드가 계산 근거는 LLMwiki/RAG에서 검색 LLM은 해석과 요약 담당 LangGraph는 흐름과 검증 담당 시황분석은 Risk-on/Risk-off 판단 후 포트폴리오 노출과 연결
